A compounding knowledge agent

This solves a problem that gets worse as you become more senior. You consume more content, but retention is low, and you connect less of it to what you already know. Most knowledge is forgotten shortly after consumption — and you keep rediscovering the same ideas again and again over the next few weeks.

A professional with the densest, most current knowledge graph in their domain will make decisions better and faster. The agent ingests YouTube videos, newsletters, and PDFs, extracts insights and concepts from each piece of content, and determines the so what for a strategy professional. It then checks every new input against everything already in the knowledge base — surfacing genuinely new information and flagging new angles on things you already know.

Top Features
🔍

Novelty scoring

Every new input is checked against the full knowledge base using semantic embeddings. Genuinely new content is surfaced. Content you already know is suppressed. New angles on familiar ideas are flagged separately — so your attention always goes to what's actually new to you.

⚡

Structured extraction with a so what

Each source is processed into INSIGHT lines (non-obvious strategic ideas) and LEARN lines (specific named concepts, tools, techniques). Every KB entry includes a so what — what a strategy professional would do differently knowing this.

📊

Topic concentration

A weekly report compares what you are actually reading against your stated learning goals — by category. It flags where you are over-indexed and where you are falling short, so your consumption changes to match your priorities, not just your interests.

✉

Morning digest

A daily email containing only new or new-angle content. Nothing you already know appears. Two sections: 🟢 New to You and 🟡 New Angles on Known Topics. Signal only — no noise.

How It Works

Content flows from four automated sources into a unified Ingest Queue. Claude extracts structured knowledge chunk-by-chunk, then scores each piece for novelty against the existing Knowledge Base using semantic embeddings. New content is staged in a Reading List before being permanently indexed.

Quality Evaluation Loop

A separate Judge LLM (Sonnet) scores every KB entry on 6 dimensions against an anchored rubric — running independently from the extraction pipeline so it can catch failures the extractor wouldn't self-report. A human-graded golden dataset (9 good/bad example pairs across all dimensions) feeds into the judge as few-shot anchors, calibrating scores to real judgments rather than abstract criteria.

Set it up for your own knowledge domain

This agent is built for AI and strategy content but the architecture works for any topic — healthcare, finance, product management, legal research. All you need is a Notion account, an Anthropic API key, and Gmail for newsletter ingestion. Setup takes about 30 minutes.

Prerequisites
🔑

Anthropic API key

Used for Claude extraction (Haiku, Sonnet, Opus). Free tier works for light use; paid recommended for daily runs. Get one at console.anthropic.com.

📄

Notion account + integration

Free Notion account. Create an internal integration at notion.so/my-integrations and copy the token. The setup script creates all three databases automatically.

✉

Gmail (for newsletters)

Enable IMAP in Gmail settings and generate an App Password (2FA required). The agent fetches newsletters matching configurable sender filters.

Step-by-Step Setup
1

Fork the repository and install dependencies

Clone the repo and install Python dependencies. Requires Python 3.10+ and Homebrew (Mac) for ffmpeg.

git clone https://github.com/surbhijain-git/Learning-Agent.git
cd Learning-Agent
pip install -r requirements.txt
brew install ffmpeg          # for YouTube Whisper fallback
2

Create your .env file

Copy .env.example to .env and fill in your keys. Required keys are marked below.

KeyRequired?What it is
ANTHROPIC_API_KEYRequiredYour Anthropic API key — used for all Claude calls
NOTION_TOKENRequiredNotion internal integration token — gives access to your workspace
NOTION_PARENT_PAGE_IDRequiredThe Notion page ID where the three databases will be created
NEWSLETTER_EMAILRequiredGmail address that receives your newsletters
NEWSLETTER_EMAIL_PASSWORDRequiredGmail App Password (not your regular password) — IMAP must be enabled
NOTION_DB_IDAuto-setNotion KB database ID — written automatically by notion_setup.py
NOTION_READING_LIST_IDAuto-setReading List DB ID — written automatically by notion_setup.py

3

Run setup to create Notion databases

This creates three Notion databases (KB, Reading List, Ingest Queue) under your parent page and writes their IDs back to your .env automatically.

python setup_check.py     # verify all keys are present
python notion_setup.py   # creates DBs and writes IDs to .env
4

Add your first content to the Ingest Queue

Open the Ingest Queue database in Notion and paste a YouTube URL or web link. The URL field is the entry point for all manual content.

To run the queue processor manually (one-shot):

python ingest_queue.py poll --once

This processes everything in the queue and writes results to the KB and Reading List.

5

Set up automation (GitHub Actions)

Add your .env keys as GitHub repository secrets, then push to main. The included workflow runs daily: newsletter fetch → Granola sync → queue poll → site data update.

Secrets needed in GitHub → Settings → Secrets and variables → Actions:

ANTHROPIC_API_KEY
NOTION_TOKEN
NOTION_PARENT_PAGE_ID
NOTION_DB_ID
NOTION_READING_LIST_ID
NOTION_INGEST_QUEUE_ID
NEWSLETTER_EMAIL
NEWSLETTER_EMAIL_PASSWORD
6

Run the eval agent to score your KB

After you have at least 10 entries in the KB, run the eval agent to score extraction quality. Results are written back to Notion and logged to logs/.

python eval/eval_agent.py spot --dry-run   # preview scores, no Notion writes
python eval/eval_agent.py spot             # score last 10 entries, write to Notion
python eval/eval_agent.py weekly           # score everything from the past 7 days
Adapting to Your Domain

Three files control what the agent extracts and what counts as valuable. Change these to adapt the agent to any knowledge domain.

📄 summariser.py — extraction prompts

The CHUNK_SYSTEM and AGG_SYSTEM prompts define what counts as an INSIGHT vs. a LEARN. For a legal research agent, you'd rewrite these to surface case precedents and statutory interpretations instead of strategic claims.

📄 eval/rubric.py — quality standards

The strategic_relevance dimension and SCORING_ANCHORS encode what "valuable" means for your use case. Rewrite the anchors to match your domain and target audience.

📄 summariser.py — novelty thresholds

The ChromaDB similarity thresholds (COVERED_THRESHOLD, RELATED_THRESHOLD) control how aggressively the agent deduplicates. Lower thresholds = more COVERED verdicts = stricter deduplication.

📄 morning_digest.py — delivery format

The digest email template, subject line framing, and section labels are all in morning_digest.py. The recipient email is a single constant at the top of the file.

AI capability is discontinuous and invisible — tasks that appear equivalent can fall on opposite sides of the competence boundary, making uniform adoption strategies dangerous. By raising baseline performance while eroding supervisory judgment, AI creates a new class of organizational vulnerability that requires intentional design choices about human-AI integration depth.
Key Claims
  • Competitive advantage accrues not from AI alone but from preserving human judgment where AI homogenizes outputs

  • Supervisory judgment degrades fastest in tasks where AI performs at 80–90% human level
Concrete Learnings
  • "Competence boundary" as a framing tool for task-level AI adoption decisions
  • Design principle: keep humans in tasks where variance is the product, not a cost
